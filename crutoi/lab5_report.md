## Theory

A **context-free grammar (CFG)** G = (VN, VT, P, S) generates a context-free language. While any CFG can describe a CFL, many algorithms (e.g., the CYK parsing algorithm) require the grammar to be in a standardised form. The most widely used is **Chomsky Normal Form (CNF)**.

A grammar is in **Chomsky Normal Form** if every production rule is of exactly one of two shapes:

1) **A → BC** — the right-hand side contains exactly two non-terminals.
2) **A → a** — the right-hand side contains exactly one terminal symbol.

If the language contains the empty string ε, an additional rule **S → ε** is permitted for the start symbol only.

The transformation from an arbitrary CFG to CNF is carried out in five ordered steps, each of which preserves the language generated by the grammar:

1) **Eliminate ε-productions** — remove all rules of the form A → ε, and add compensating rules for every production that contained a nullable non-terminal.

2) **Eliminate unit (renaming) productions** — remove rules of the form A → B where both sides are single non-terminals, replacing them with the transitive closure of what B can derive.

3) **Eliminate inaccessible symbols** — remove any non-terminal that can never be reached from the start symbol.

4) **Eliminate non-productive symbols** — remove any non-terminal that cannot derive any string of terminals.

5) **Convert to CNF** — replace each terminal in a mixed or long production with a dedicated terminal non-terminal (e.g. `Ta → a`), then binarize every production of length ≥ 3 by introducing helper non-terminals.

## Normalisation – Step by Step

### Step 1: Eliminate ε-Productions

**Find nullable symbols** (those that can derive ε directly or transitively):

- Rule 9: `B → ε` → **B is nullable**.
- No other non-terminal can derive ε through B alone.

**Nullable set: {B}**

For every production that contains B, add a copy with B removed:

| Original rule | Added rule |
|---------------|------------|
| S → abA**B**  | S → abA    |
| A → **B**S    | A → S      |
| B → **B**A    | B → A      |
| B → abab**B** | B → abab   |

Remove rule 9 (`B → ε`).

**After Step 1:**
```
S → abAB        S → abA
A → aSab        A → BS      A → S (unit!)
A → aA          A → b
B → BA          B → A  (unit!)
B → ababB       B → abab    B → b
C → AS
```

---

### Step 2: Eliminate Unit Productions

A **unit production** has a single non-terminal on the right-hand side.

Unit productions found: `A → S` and `B → A`.

**Compute unit closures:**

| Variable | Unit closure |
|----------|-------------|
| S        | {S}         |
| A        | {A, S}      |
| B        | {B, A, S}   |
| C        | {C}         |

For each variable X, keep all non-unit productions derivable from every member of unit(X):

- **A** inherits S's non-unit productions: `A → abAB`, `A → abA`
- **B** inherits A's non-unit productions: `B → aSab`, `B → BS`, `B → aA`, `B → b`, `B → abAB`, `B → abA`
- **B** inherits S's non-unit productions: already covered by the above

Remove `A → S` and `B → A`.

**After Step 2:**
```
S → abAB        S → abA
A → aSab        A → BS      A → aA      A → b
A → abAB        A → abA
B → BA          B → ababB   B → abab    B → b
B → aSab        B → BS      B → aA
B → abAB        B → abA
C → AS
```

---

### Step 3: Eliminate Inaccessible Symbols

Starting from S, find all reachable non-terminals by following every production:

- S → abAB → **A**, **B** reachable
- A → BS, A → aSab, … → S, B reachable (already)
- B → BA, B → ababB, … → B, A, S reachable (already)

**Reachable: {S, A, B}**  
**Inaccessible: {C, D}** — remove `C → AS` and `D` (no productions).

**After Step 3:**
```
S → abAB        S → abA
A → aSab        A → BS      A → aA      A → b
A → abAB        A → abA
B → BA          B → ababB   B → abab    B → b
B → aSab        B → BS      B → aA
B → abAB        B → abA
```

---

### Step 4: Eliminate Non-Productive Symbols

A symbol is **productive** if it can derive at least one string of terminals.

- `A → b` → A is productive.
- `B → b` → B is productive.
- `S → abA` (terminals + productive A) → S is productive.

**All remaining symbols {S, A, B} are productive.** Nothing to remove.

**After Step 4:** same as after Step 3.

---

### Step 5: Convert to Chomsky Normal Form

**Step 5a – Introduce terminal non-terminals**

Replace every terminal in productions of length > 1 with a dedicated non-terminal:

```
Ta → a
Tb → b
```

**Step 5b – Binarize long productions (right-fold)**

Productions of length ≥ 3 are folded right-to-left. A pair cache ensures the same right-hand pair is never introduced twice:

| Helper | Production   | Represents (string) |
|--------|--------------|---------------------|
| N1     | N1 → A B     | `AB`                |
| N2     | N2 → Tb N1   | `bAB`               |
| N3     | N3 → Tb A    | `bA`                |
| N4     | N4 → Ta Tb   | `ab`                |
| N5     | N5 → S N4    | `Sab`               |
| N6     | N6 → Tb B    | `bB`                |
| N7     | N7 → Ta N6   | `abB`               |
| N8     | N8 → Tb N7   | `babB`              |
| N9     | N9 → Tb N4   | `bab`               |

**How each original production maps to CNF:**

| Original         | CNF          | Explanation |
|------------------|--------------|-------------|
| S → abAB         | S → Ta N2    | Ta·(Tb·(A·B)) — N1=AB, N2=Tb·N1 |
| S → abA          | S → Ta N3    | Ta·(Tb·A) — N3=Tb·A |
| A → aSab         | A → Ta N5    | Ta·(S·(Ta·Tb)) — N4=Ta·Tb, N5=S·N4 |
| A → BS           | A → BS       | already CNF |
| A → aA           | A → Ta A     | binary after terminal sub |
| A → b            | A → b        | already CNF |
| A → abAB         | A → Ta N2    | same as S → abAB |
| A → abA          | A → Ta N3    | same as S → abA |
| B → BA           | B → BA       | already CNF |
| B → ababB        | B → Ta N8    | Ta·(Tb·(Ta·(Tb·B))) — N6=Tb·B, N7=Ta·N6, N8=Tb·N7 |
| B → abab         | B → Ta N9    | Ta·(Tb·(Ta·Tb)) — N4=Ta·Tb, N9=Tb·N4 |
| B → b            | B → b        | already CNF |
| B → aSab         | B → Ta N5    | same as A → aSab |
| B → BS           | B → BS       | already CNF |
| B → aA           | B → Ta A     | binary after terminal sub |
| B → abAB         | B → Ta N2    | same as S → abAB |
| B → abA          | B → Ta N3    | same as S → abA |

## Implementation Description

### `Grammar` Class (`lab5_cnf.py`)

All five normalisation steps are encapsulated as methods of a single `Grammar` class. The constructor accepts any grammar, satisfying the bonus requirement:

```python
class Grammar:
    def __init__(self, non_terminals, terminals, productions, start):
        self.non_terminals = set(non_terminals)
        self.terminals = set(terminals)
        self.productions = {nt: [list(p) for p in prods]
                            for nt, prods in productions.items()}
        self.start = start
        self._helper_count = 0
```

Productions are stored as `dict[str, list[list[str]]]`. ε is represented as the single-element list `['e']`.

---

### Step 1 – `eliminate_epsilon()`

Iteratively finds nullable symbols, then for each production generates every combination with nullable symbols optionally removed using a bitmask over their positions:

```python
npos = [i for i, s in enumerate(prod) if s in nullable]
for mask in range(1 << len(npos)):
    new_prod = list(prod)
    for bit, pos in enumerate(npos):
        if mask & (1 << bit):
            new_prod[pos] = None
    new_prod = [s for s in new_prod if s is not None]
    if new_prod and new_prod not in new_prods[nt]:
        new_prods[nt].append(new_prod)
```

---

### Step 2 – `eliminate_unit_productions()`

Computes the unit closure of each non-terminal using BFS, then replaces unit productions with all reachable non-unit productions:

```python
def unit_closure(nt):
    visited, queue = {nt}, [nt]
    while queue:
        cur = queue.pop()
        for prod in self.productions.get(cur, []):
            if len(prod) == 1 and prod[0] in self.non_terminals \
                    and prod[0] not in visited:
                visited.add(prod[0])
                queue.append(prod[0])
    return visited
```

---

### Step 3 – `eliminate_inaccessible()`

BFS from the start symbol over the production graph. Any non-terminal not reached is removed together with its rules.

---

### Step 4 – `eliminate_non_productive()`

Iteratively marks symbols as productive: initially all terminals, then any non-terminal whose at least one production has every symbol already productive.

---

### Step 5 – `to_cnf()`

Two sub-passes:

1. **Terminal substitution** — replaces every terminal in a production of length > 1 with a helper `Tt → t`, using a cache to avoid duplicates.

2. **Binarization** — right-folds every production of length > 2 by replacing the last two symbols with a cached helper non-terminal:

```python
def binarize(prod):
    prod = list(prod)
    while len(prod) > 2:
        pair = (prod[-2], prod[-1])
        if pair not in pair_cache:
            helper = self._new_helper()
            self.non_terminals.add(helper)
            self.productions[helper] = [list(pair)]
            pair_cache[pair] = helper
        prod = prod[:-2] + [pair_cache[pair]]
    return prod
```

The `pair_cache` guarantees that identical right-hand pairs share a single helper, keeping the grammar compact.

## Conclusions

1. The five-step normalisation procedure systematically eliminates each class of non-CNF rule while preserving the generated language. Performing the steps in the prescribed order is critical — for instance, eliminating ε-productions before unit productions prevents the two from interfering.
2. The **bitmask enumeration** in Step 1 cleanly handles grammars where multiple nullable symbols appear in the same production, generating all 2^k subsets in O(2^k) time without nested loops.
3. **Unit closure via BFS** in Step 2 correctly handles transitive chains (B → A → S) in a single pass, avoiding repeated scans over the production set.
4. Steps 3 and 4 are not merely cosmetic — removing inaccessible and non-productive symbols reduces the grammar size, which directly improves the O(n³ |G|) complexity of CYK parsing.
5. The **pair cache** in Step 5 is essential: without it, the same binary pattern `(X, Y)` would generate redundant helper rules. With it, every distinct pattern maps to exactly one helper, keeping the CNF grammar compact.
6. The `Grammar` class satisfies the bonus requirement by accepting any CFG as input — changing the grammar in the `__main__` block is sufficient to convert any variant's grammar without modifying the implementation.
